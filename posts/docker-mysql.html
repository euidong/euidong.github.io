Mysql

Mysql is a basic Database to store countless data. It is RDMS(This think that Relation is very very very important than any other things). You can Read, Create, Update, Delete data with it faster, managable (like transaction) than FileSystem. It is manage by Oracle. And It can get good performance in read.

Espectially 5.7 version is stable version. So industry use this version for development or test.

Docker

Docker is a virtualization application. It is usally show more performance than any other virtualization machine program. Becase It use container. So, It can make more efficient feature. If you want get database running in Linux then you just build and run container that has database with Linux. When you anymore need taht database, you just delete that container. It's so simple, clear. Let's enjoy.

Mysql with Docker

Just Run script

1sudo docker run \ # run container
2 -e MYSQL_ROOT_PASSWORD=qwe123 \ # setting MYSQL_ROOT_PASSWORD
3 -e MYSQL_DATABASE=just \ # default database name
4 -e TZ=Asia/Seoul \ # setting Timezone
5 -p 3309:3306 \ # setting port (my Pc port:container port)
6 mysql:5.7 \ # image name
7 --character-set-server=utf8 \ # setting utf8
8 --collation-server=utf8_unicode_ci # setting utf8

With compose file

docker-compose.yml

1version: "3.7"
2services:
3  # Mysql Server
4  db:
5    image: mysql:5.7
6    # mysql utf-8 설정
7    command:
8      - --character-set-server=utf8
9      - --collation-server=utf8_unicode_ci
10    # db password 및 db 자동 생성
11    environment:
12      MYSQL_ROOT_PASSWORD: "qwe123"
13      MYSQL_DATABASE: "just"
14      MYSQL_TCP_PORT: "3306"
15      TZ: "Asia/Seoul"
16    ports:
17      - "3309:3306"
18    # for db data backUp
19    volumes:
20      - ./mysql/data:/var/lib/mysql
1docker-compose up --build

more information => https://docs.docker.com/get-started/
